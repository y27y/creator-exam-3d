# 系统架构文档

## 造物者考核3D 系统总览

本文档描述游戏的24个核心系统模块及其交互关系。

---

## 一、核心引擎层

### 1. GameEngine (gameEngine.js)
**职责**：游戏核心逻辑、状态管理、回合结算
**关键功能**：
- 回合制游戏循环（玩家回合 → 环境结算 → 单位移动 → 胜负判定）
- 资源管理（奇迹点MP、造物次数、裂隙值Entropy）
- 单位管理（平民、使者、巨兽、部落）
- 灾害扩散算法（BFS + 动态权重）
- A*寻路（时间感知 + LRU缓存）
- 地形系统（临时地形、地形恢复）

**接口**：
- `placeCreation(card, x, y)` - 放置造物
- `endTurn()` - 结束回合
- `getTerrain(x, y)` - 获取地形
- `distance(a, b)` - 计算距离

### 2. CreatorExam3D (game.js)
**职责**：3D渲染、UI交互、用户输入处理
**关键功能**：
- Three.js 3D场景管理
- 射线检测（格子选择）
- 粒子特效触发
- 屏幕特效（闪光、震动、暗角）
- UI更新（状态面板、日志、手牌）
- 动画系统（单位移动、造物放置）

**继承**：extends GameEngine

### 2.1 长夜守城模式 (public/modes/tower-defense/)
**职责**：第6到第7天之间的塔防过场、造物防线化、守夜结果回写
**关键功能**：
- 通过 `creatorExamNightWatchContext` 接收主游戏状态，包含熵值、居民、最多18张最近造物卡、近期世界事件经历和玩家风格
- `towerBridge.js` 负责主游戏上下文、守夜旁白、波次通讯和结果写回；`nightWatchTowers.js` 负责前端塔计划应用、数值偏向、塔池筛选和普通放置上限移除
- 后端 `/api/night-watch-towers` 使用 DeepSeek Flash 等 OpenAI-compatible 模型生成守夜塔名称、描述、EX说明、颜色、因果简报、三选一守夜加成和数值偏向；无密钥或失败时使用 `server/nightWatchTowers.js` 本地兜底
- 塔的底层攻击/支援逻辑继续复用原有塔类型，AI只改变名称、说明、选择池和已白名单校验的数值倍率
- 开局简报展示“此前行为 -> 守夜获得内容”，三选一加成只允许白名单效果（开局金币/生命、核心塔伤害/射程/折扣、周期补给），AI不直接生成可执行逻辑
- 普通 `limit` 在守夜模式中移除；`exLimit` 保留，EX升级仍按原计数限制执行
- 结算写入 `creatorExamNightWatchResult`，主游戏收到后记录 `defense_resolved` 世界事件，并把守夜塔计划摘要带给后续空域和结局

**边界**：
- 不把大型塔防 HTML 一次性拆入主引擎；当前只拆出守夜塔生成/应用模块，战斗循环仍留在独立模式页
- 不允许AI返回新塔类型或任意执行逻辑；所有类型、数值范围和颜色都经过白名单/夹取校验
- 不在每帧或每次攻击时调用AI；AI只在入场生成塔计划、波次通讯和结算文本时低频使用

### 2.2 第七天空域模式 (public/modes/air-combat/)
**职责**：终局空战清算层、造物武器化、结果回写
**关键功能**：
- 通过 `creatorExamAirCombatContext` 接收主游戏状态（熵值、居民、最近造物、守夜结果）
- 将造物能力映射为 Canvas 空战武器与造物脉冲
- 通过 `/api/narrative` 生成空域简报、Boss台词、武器副作用、近身通讯和胜负清算，DeepSeek Flash 等模型可由后端环境变量切换
- 把实时射击、敌机、弹幕、Boss、碰撞、道具和分数限制在独立模式页
- 将 6 段 Boss 航线结算为 `creatorExamAirCombatResult`
- 主游戏收到结果后写入 `airspace_resolved` 世界事件，并生成最终结局钩子

**边界**：
- 不迁入外部空战仓库的联机、排行榜、挑战码、普通关卡菜单
- 不把实时战斗逻辑放入 `GameEngine` 回合制核心
- 不在每帧或每次射击时调用AI；AI只进入低频叙事节点，实时数值、碰撞、胜负保持本地可验证

### 3. DebugGame (debug/debugGame.js)
**职责**：CLI调试工具、自动化测试
**关键功能**：
- 命令行游戏控制台
- 所有系统的集成实例
- 测试钩子（覆盖方法用于测试）
- 地图打印、日志打印、统计输出

**继承**：extends GameEngine

## 二、AI与叙事层

### 4. AIClient (aiClient.js)
**职责**：自然语言造物编译、AI API调用
**关键功能**：
- 自然语言 → 结构化卡牌（AI解析 + 本地兜底）
- 创意评分系统（诗意词汇、修辞手法、直接能力词检测）
- 能力标签映射（关键词 → 能力类型）
- 本地编译器（正则 + 模板匹配）

**接口**：
- `compileCreation(text)` - 编译自然语言为卡牌
- `localCompile(text)` - 本地兜底编译
- `calculateCreativityScore(text, card)` - 计算创意分

### 5. Storyteller (storyteller.js)
**职责**：AI故事叙述者、动态事件生成
**关键功能**：
- 4种AI人格：卡珊德拉（悲观）、菲比（乐观）、兰迪（混沌）、说书人（平衡）
- 动态事件频率调整（基于人格）
- 与aiMemory深度联动
- 6种结局分支判定

**接口**：
- `generateEvent(gameState)` - 生成随机事件
- `determineEnding(gameState)` - 判定结局类型
- `getNarrativeSummary()` - 获取叙事摘要

### 6. AIMemory (aiMemory.js)
**职责**：玩家行为追踪、个性化叙事
**关键功能**：
- 行为追踪（造物偏好、救援成功率、资源使用）
- 玩家画像生成（用于史诗结局）
- 跨会话持久化（localStorage）
- 游戏风格分析（自然派/造物狂/完美主义者/牺牲者/平衡型）

### 7. WorldLegend (worldLegend.js)
**职责**：涌现式世界传说生成
**关键功能**：
- 生成式神话语法（CFG上下文无关文法）
- 4种原型：创世、毁灭、轮回、混沌
- 动态传说演化（基于玩家行为）
- 传说碎片收集系统

### 8. CausalGraph (game.js 内嵌)
**职责**：叙事因果引擎
**关键功能**：
- 蝴蝶效应系统（微小选择 → 长期影响）
- 因果链追踪
- 叙事债务检测
- 多世界线支持

## 三、NPC与社交层

### 9. NPCManager (npcManager.js)
**职责**：NPC管理、情绪系统、环境叙事
**关键功能**：
- 8种情绪状态（希望、恐惧、信任、愤怒、悲伤、敬畏、困惑、决心）
- 情绪传染机制
- 动态特质系统（trust/fear/hope）
- 游戏事件响应（rescue/win/loss/hazard）
- 地图位置参与（单位遇到NPC获得增益）

### 10. SocialGraph (socialGraph.js)
**职责**：NPC关系网络
**关键功能**：
- 6种关系类型（友谊、竞争、师徒、敌对、同盟、暗恋）
- 关系强度动态演化
- 小团体检测（clique detection）
- 社交距离计算
- 情绪传染网络

### 11. LegacySystem (legacySystem.js)
**职责**：单位传承系统（Wildermyth风格）
**关键功能**：
- 8种特质（勇敢、谨慎、智慧、强壮、敏捷、幸运、领导力、牺牲精神）
- 4个传承等级（学徒→守护者→英雄→传说）
- 救援历史记录
- 跨关卡属性继承
- 特质组合效果

### 12. PersistentWorld (persistentWorld.js)
**职责**：跨关卡持久化
**关键功能**：
- 被救援单位在未来关卡重现
- 关系状态跨关卡继承
- 世界状态导出/导入
- 关卡间因果连接

## 四、策略深度层

### 13. ChainReactionCodex (chainReactionCodex.js)
**职责**：连锁反应系统
**关键功能**：
- 13种共鸣组合（光+水=蒸汽、土+水=沼泽等）
- 图鉴系统（发现追踪、使用统计）
- 粒子特效联动
- 共鸣条件检测（距离、类型匹配）

### 14. RitualForge (ritualForge.js)
**职责**：仪式熔炉系统
**关键功能**：
- 8种配方（光之仪式、水祭、大地契约等）
- 4种原型（秩序、混沌、生命、虚空）
- 仪式发现机制（实验解锁）
- 涌现式仪式（非配方组合产生意外效果）
- 仪式失败与腐化风险

### 15. OathManager (oathbinding.js)
**职责**：言灵誓约系统
**关键功能**：
- 5种誓约（守护、知识、复仇、牺牲、血缘）
- 背叛概率计算（基于声誉、压力、诱惑）
- 6种背叛触发器
- 怨恨链机制（背叛引发连锁报复）
- 声誉系统

### 16. CognitiveAbyss (cognitiveAbyss.js)
**职责**：认知深渊系统
**关键功能**：
- 高熵游戏层（混沌模式）
- 谜题解码（凯撒密码、反向文本、符号替换）
- NPC反转（敌人变友军、友军变敌人）
- 悖论卡牌（非法能力产生意外效果）
- 深渊深度系统

### 17. VerificationCorruption (verificationCorruption.js)
**职责**：验证腐化系统
**关键功能**：
- 悖论卡牌（非法能力产生腐化）
- 引擎过载（过多造物导致系统不稳定）
- 腐化传播（错误扩散影响其他造物）
- 净化机制（消耗资源修复）
- 腐化等级系统（轻度→中度→重度→临界）

### 18. CreatorWorkshop (creatorWorkshop.js)
**职责**：造物者工坊系统
**关键功能**：
- 4种操作（拆解、改造、融合、注入）
- 8种材料（灵木、岩心、源水、光尘、影纱、时砂、忆晶、混沌碎片）
- 6种改造（范围扩展、持久化、精简、稳固、过载、回响绑定）
- 融合配方表
- 品质系统（普通→精炼→大师之作→腐化）

### 19. EnemyIntentSystem (enemyIntent.js)
**职责**：敌方意图预览系统
**关键功能**：
- 4类实体意图预测（巨兽、平民、使者、灾害）
- 威胁等级系统（无/低/中/高）
- 置信度评分（0-1）
- 战略建议生成
- 战术叙事生成

## 五、记忆与效果层

### 20. VectorMemory (vectorMemory.js)
**职责**：语义向量记忆系统
**关键功能**：
- 50维TF-IDF向量表示
- 余弦相似度语义检索
- 记忆衰减机制（时间衰减 + 重要性加权）
- 与裂隙回响系统联动

### 21. CognitiveEffects (cognitiveEffects.js)
**职责**：认知效果系统
**关键功能**：
- 文本扭曲（模拟记忆混乱）
- 矛盾对话（NPC说反话）
- 记忆碎片（闪回效果）
- 既视感（deja vu）

### 22. RiftEchoSystem (riftEchoes.js)
**职责**：裂隙回响系统
**关键功能**：
- 4种回响类型（仁慈、恶意、悖论、预言）
- 过去造物的幽灵残留
- 能力反转映射
- VectorMemory集成（语义相似度触发）
- 动态回响类型（基于腐化度和深渊深度）

## 六、辅助层

### 23. AbilityHandlers (abilityHandlers.js)
**职责**：能力处理策略模式
**关键功能**：
- 20+种能力的策略实现
- 3层消耗体系（MP + Entropy + Stability）
- 能力验证（语法→语义→叙事）
- 副作用处理

### 24. ValidationEngine (validationEngine.js)
**职责**：造物验证引擎
**关键功能**：
- 3层验证（语法、语义、叙事）
- 错误分级（警告/错误/致命）
- 自动修复建议
- 与AIClient联动

### 25. Particles (particles.js)
**职责**：粒子特效系统
**关键功能**：
- 能力激活粒子（按类型定制）
- 救援/失败特效
- 浮动文字
- 环境粒子（雨、雪、雾）
- 屏幕特效（闪光、震动、暗角）

### 26. Levels (levels.js)
**职责**：关卡配置
**关键功能**：
- 6个关卡配置（洪水村庄、永夜矿井、巨兽困城、失语战争、记忆瘟疫、创世终考）
- 地形符号映射
- 单位初始配置
- 灾害参数
- 关卡克隆（深拷贝）

## 数据流

### 1. 造物流程
```
玩家输入自然语言
  → AIClient.compileCreation()
    → AI API调用（失败则localCompile兜底）
      → 生成结构化卡牌（ability, range, duration, cost, stabilityCost）
        → ValidationEngine.validate() 验证
          → GameEngine.placeCreation() 放置
            → AbilityHandlers 执行效果
              → ChainReactionCodex 检测共鸣
                → Particles 触发特效
                  → AIMemory 记录行为
                    → Storyteller 评估叙事影响
```

### 2. 回合结算流程
```
GameEngine.endTurn()
  → 造物持续时间衰减
    → 灾害扩散（BFS + 权重）
      → 单位移动（A*寻路）
        → NPCManager.reactToGameEvent() NPC响应
          → SocialGraph 传播情绪
            → EnemyIntentSystem.generatePreviews() 生成意图预览
              → 胜负判定
                → 持久化状态
```

### 3. 叙事生成流程
```
游戏事件触发
  → Storyteller.generateEvent()
    → AIMemory.getProfileSummary() 获取玩家画像
      → WorldLegend.getRelevantLore() 获取相关传说
        → CausalGraph.getConsequences() 获取因果影响
          → 生成叙事文本
            → CognitiveEffects.apply() 应用认知效果（高熵时）
              → RiftEchoSystem.checkEchoes() 检测裂隙回响
```
