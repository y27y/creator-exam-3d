# 飞机大战融合构思：第七天裂隙空域

日期：2026-07-05

## 一句话定位

飞机大战不是额外小游戏，而是《造物者考核3D》的“清算层”：玩家在地面创造世界、在长夜守住世界，最后在第七天进入裂隙空域，清除所有残留的灾害、失败、记忆和敌意。

## 放在什么地方

推荐放在第七天。

当前主线已经多次使用“第七天到来时”作为结局锚点，并且最后一关是“终考：第七天之前”。飞机大战最适合作为第七天真正到来时的终局玩法。

主线顺序建议：

```text
第1-5天：学习创造、救援、理解灾害
第6天：终考，证明能让造物形成秩序
第6天夜晚：塔防，证明能守住秩序
第7天：飞机大战，进入裂隙空域完成最终清算
```

建议命名：

- 第七天：裂隙空域
- 空域终考
- 创世者航迹
- 深层裂隙巡航

## 核心叙事理由

造物者不是突然驾驶飞机，而是把自己的造物、记忆和世界裂隙压缩成一艘“空域载体”。在裂隙空域里，灾害不再表现为格子、水、雾、战争，而是表现为弹幕、敌机、Boss 阶段和高速压力。

地面策略层回答：“你创造了什么？”

空战清算层回答：“这些创造在失控的裂隙里能不能活下来？”

## 采用范围

飞机大战仓库主要提供实时射击、敌机、弹幕、碰撞、道具、Boss 与无尽推进逻辑。第一阶段不要把精力放在重写这些战斗逻辑上，而是借用它稳定的运行骨架，把外壳、关卡结构和叙事驱动改成《造物者考核3D》的第七天空域。

原仓库的普通关卡可以直接丢弃或只作为素材参考。真正要保留的是无尽模式的循环能力，但它在本项目里不应表现为“真的无尽”，而应被改造成 6 到 7 个 Boss 段落的终局航线。

优先采用：

- 无尽模式的生成循环
- Boss、精英敌、阶段推进
- 玩家火力、敌机、弹幕、道具
- 碰撞、血量、结算和分数系统

需要改造：

- 入口从“单独游戏”改为“第七天空域”
- 删除原始关卡菜单、公告、外部排行榜等独立游戏外壳
- 把无尽模式切成 6 到 7 个 Boss 段落
- 武器来源接入主游戏造物
- 敌人和关卡主题接入世界状态
- 胜败结果回写主世界

暂缓采用：

- 重写成 3D
- 把空战直接嵌入 7x7 地图
- 让空战每一秒都调用 AI

## 与主游戏怎么连接

主游戏传入飞机大战的上下文：

```js
{
  mode: 'rift_airspace',
  day: 7,
  endingPressure,
  entropy,
  rescuedResidents,
  lostResidents,
  recentCreations,
  towerDefenseResult,
  discoveredLore,
  playerStyle
}
```

飞机大战回传主游戏的结果：

```js
{
  outcome: 'victory',
  score,
  clearedLayers,
  bossDefeated,
  damageTaken,
  creationOverload,
  rescuedEchoes,
  endingModifier
}
```

回传结果用于决定最终结局文本、世界传说、居民后续状态和是否开放无尽巡航。

## 玩法映射

| 主游戏元素 | 飞机大战表现 |
|---|---|
| 自然语言造物 | 主武器、副武器、护盾、僚机、主动技能 |
| 造物稳定度 | 武器过热、冷却、失控概率 |
| 熵值 | 弹幕密度、敌人变异、Boss 阶段 |
| 已救居民 | 僚机、通讯员、救援回声 |
| 已失去居民 | 记忆敌人、幻影弹幕、特殊剧情压力 |
| 世界传说 | 关卡标题、Boss 名称、空域事件 |
| 塔防结果 | 第七天空域初始难度和奖励 |
| AI Storyteller | 动态事件、Boss 台词、波次主题 |

## 模式设计

### 第七天空域主线

固定出现在主线末尾。不要沿用飞机大战原仓库的普通关卡结构，而是把无尽模式改造成一条有终点的 Boss 航线。

目标：

- 完成 6 到 7 个空域段落
- 每个段落以一个 Boss、精英潮或叙事检查点收束
- 前 6 个 Boss 分别对应主线前六天/六关留下的灾害、失败、造物副作用或居民记忆
- 第 7 个 Boss 作为第七天裂隙核心，可按剧情需要保留或放到真结局路线
- 根据表现影响最终结局

推荐结构：

```text
Boss 1：洪水残响
Boss 2：迷雾航标
Boss 3：战争回声
Boss 4：巨兽阴影
Boss 5：居民誓约
Boss 6：终考秩序
Boss 7：第七天裂隙核心（可选真结局）
```

### 空域支线

可选短关。用于在主线中后段提前展示空战系统，但不建议太早开放完整模式。

用途：

- 拦截一次裂隙风暴
- 护送居民迁移
- 净化某个区域的高空灾害

### 深层裂隙巡航

通关后再开放真正的无尽巡航。主线阶段不要让玩家误以为这是无尽刷分模式，而应明确它是第七天终局清算。

用途：

- 分数挑战
- 刷稀有材料
- 测试造物武器组合
- 让 AI 根据世界记忆生成变异波次

## AI 驱动点

AI 驱动是这个模式的主体，不是 UI 上的点缀。空战仓库负责实时手感和可玩性，AI 负责把每个 Boss、每段航线、每次通讯和结算变成主游戏世界记忆的清算。

原则：AI 写剧情、解释、身份、台词、变体和结算；本地规则负责实时战斗、数值和胜败。

可生成内容：

- 空域层名称
- Boss 叙事身份、登场台词、阶段转换台词
- 敌人波次主题和变异解释
- 玩家造物武器的描述和副作用
- 通讯员台词、居民回声、失去居民的幻听
- 结算传说、最终结局修饰
- Boss 与玩家造物之间的短句互动
- 根据玩家风格生成的“空域裁决词”

空战中应加入的小 trick：

- Boss 进入新阶段时说一句与前六天记忆相关的话。
- 敌机或裂隙残影靠近玩家/僚机时触发一句短台词。
- 玩家使用由造物转化来的武器时，AI 生成一次“造物副作用”旁白。
- 击败 Boss 后显示一张短 CG/PPT 式清算卡，而不是直接进入下一段战斗。
- 通讯员可以由已救居民担任；如果居民死亡或失踪，则改为回声/幻听。

必须本地规则控制：

- 碰撞
- 弹幕
- 血量
- 掉落
- 分数
- 胜败
- 结局分支条件

必须避免：

- 每一帧或每次射击调用 AI。
- 让 AI 直接修改血量、碰撞、掉落概率等实时数值。
- 把 AI 文案做成单独说明面板，和战斗过程脱节。
- 保留原仓库“普通飞机大战”的菜单语气、关卡名、公告、外部登录或排行榜。

## 塔防融合经验迁移到空战

塔防融合暴露出的关键问题：如果只是把另一个游戏页面嵌进来，即使逻辑能跑，也会显得像两个游戏拼在一起。空战必须从一开始避免这个问题。

可复用经验：

1. 桥接层优先
   不直接重写原游戏核心逻辑，先加 `airCombatBridge.js` 接收主游戏上下文、改标题/按钮/文案、注入 AI 叙事、回传结算。

2. 删除独立游戏外壳
   原仓库里的公告、登录、外部排行榜、普通关卡选择、测试季文案都应删掉或隐藏。玩家看到的只能是《造物者考核3D》的空域终局。

3. 进入玩法前必须有过场
   不能点按钮后直接跳进空战菜单。需要先出现 2 到 4 页 PPT/CG 式过场，说明“第六关之后、第七天到来、造物被压缩为空域载体”。

4. 关卡不是照搬，而是重排
   塔防的 30 波被重解释成 6 夜守城；空战也应把无尽模式重解释成 6 到 7 个 Boss 段落，而不是沿用原仓库关卡。

5. AI 要嵌入战斗瞬间
   只在开头和结尾放 AI 文案不够。应有 Boss 阶段台词、敌人近身短句、武器副作用旁白、击败后清算卡。

6. 本地兜底必须完整
   所有 AI 文案都要有本地句库兜底。没有 API key 时，空战仍然完整可玩，只是动态文本减少。

7. 测试要防止“旧壳回流”
   加源码级回归测试，检查是否仍存在原仓库公告、外部排行榜、普通关卡菜单、非主游戏命名等。

## 造物武器化

玩家在主游戏里的造物可以被压缩为空战装备。

例子：

```text
会发光并指路的月亮树
-> 月树光束：直线穿透弹，短暂揭示隐形敌人

吸走洪水的透明鲸鱼
-> 鲸潮护盾：吸收部分弹幕并转化为能量

连接梦境的桥
-> 梦桥僚机：复制玩家移动轨迹并延迟开火
```

第一阶段只需要做 5 到 8 个映射模板，不需要支持所有造物自由变形。

## 技术接入方式

飞机大战来自 GitHub 仓库，建议先作为独立模式迁入：

```text
public/modes/air-combat/
  index.html
  airCombatBridge.js
  src/
  assets/
```

接入步骤：

1. 先拉起原仓库，确认无尽模式、Boss、敌机、弹幕、道具、结算能独立运行。
2. 删除或旁路原普通关卡入口，确定主线只走改造后的有限无尽模式。
3. 增加 `airCombatBridge.js`，接收主游戏上下文。
4. 增加第七天空域过场，解释第六关到第七关之间为什么进入空战。
5. 做一个“造物到武器”的模板映射层。
6. 把无尽模式切成 6 到 7 个 Boss 段落。
7. 让 Boss、敌潮和通讯读取 `entropy`、`playerStyle`、`recentCreations`、`towerDefenseResult`。
8. 结算后回传结果给主游戏。

桥接层职责：

- 读取 `creatorExamAirCombatContext`。
- 改写页面标题、按钮、模式名、Boss 名称和结算文案。
- 注入 AI 过场、Boss 台词和通讯员短句。
- 提供本地 fallback 文案。
- 发布 `creatorExamAirCombatResult`。

原空战核心职责：

- 保持输入、移动、射击、碰撞、弹幕、掉落、Boss 行为和帧循环稳定。
- 提供可被桥接层配置的 Boss 列表、阶段事件和结算钩子。

## 第一阶段验收

- 能从主游戏或临时入口打开飞机大战。
- 能读取至少 3 个主游戏状态：`entropy`、`recentCreations`、`rescuedResidents`。
- 至少 1 个主游戏造物能变成空战武器。
- 普通关卡选择不出现在主线入口里。
- 主线空战不是无限刷分，而是 6 到 7 个 Boss 段落。
- 进入空战前有第七天空域过场。
- 至少 3 个 Boss 使用前六天/六关记忆重新命名和包装。
- 至少 1 个 Boss 阶段触发 AI 或本地 fallback 台词。
- 至少 1 个敌人近身/武器释放/击败节点触发战斗内短句。
- 通关或失败后能回传 `outcome`、`score`、`endingModifier`。
- 主游戏能把结果写入 `airspace_resolved` 世界事件。

## 关键风险

- 主题突兀：必须坚持“裂隙空域”和“造物压缩”的设定，避免变成菜单里的普通飞机大战。
- 原关卡包袱：不要为了尊重原仓库而保留普通关卡流程；那会削弱主线节奏。
- 实时玩法抢主线：空战应该成为终局高潮和可选无尽，不要替代主游戏的造物解谜。
- 仓库依赖未知：等拿到 GitHub 仓库后，需要先判断是直接迁移、保留构建流程，还是抽核心逻辑。
- AI 驱动过度：实时战斗不能依赖网络 AI，否则延迟和不稳定会破坏手感。
- AI 驱动不足：如果只在开头/结尾生成文案，比赛核心亮点会不明显。AI 必须进入 Boss、通讯、武器和结算节点。

## 暂定优先级

中高。飞机大战适合作为第七天终局和通关后无尽玩法。建议在塔防完成最小桥接后接入，这样主线节奏会更自然：先守住地面，再打穿天空。
