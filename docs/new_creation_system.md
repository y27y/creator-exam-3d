# 新造物系统设计文档

## 设计目标
1. 选择性丰富：每个关卡有多种通关策略
2. 不重复：每种能力有独特玩法
3. 有策略深度：能力之间存在协同和克制
4. 有代价感：强力造物需要承担风险

## 新机制

### 1. 环境共鸣（Resonance）
当两个同系造物距离<=2时，产生共鸣效果：
- 水系共鸣：吸水范围+1
- 光系共鸣：照明持续回合+1
- 地形系共鸣：改造的地形不会随造物消失而恢复
- 防御系共鸣：屏障耐久+1（可多挡一次）
- 精神系共鸣：引导范围+1

### 2. 连锁反应（Chain Reaction）
某些能力组合会产生额外效果：
- 照明 + 吸水：被照亮的水域更容易被吸收（吸水范围+1）
- 森林 + 屏障：森林中的屏障不可被破坏
- 引导 + 记忆信标：被引导的单位免疫记忆混乱
- 陷阱 + 迟缓：陷阱中的巨兽额外停留1回合
- 冻结 + 抬升：冻结后抬升的地形永久存在

### 3. 副作用选择（Side Effect Choice）
玩家可以在放置时选择"强化模式"：
- 普通模式：标准效果，标准副作用
- 强化模式：效果+50%，但副作用翻倍
- 危险模式：效果翻倍，但立即增加2裂隙

### 4. 地形互动（Terrain Interaction）
- 水域 + 照明 = 产生蒸汽迷雾（临时阻挡视野，但可通行）
- 森林 + 洪水 = 森林被淹没变成沼泽
- 高地 + 黑暗 = 黑暗无法覆盖高地
- 陷阱 + 水域 = 陷阱失效

## 关卡调整建议

### 第1关：洪水村庄
**问题**：不造任何东西就能通关
**修改**：
- 地图：增加中间水域带
```
~~~....
~~..~~~
~..V..H
n~~V~~H
n~.V..H
n~~~...
~~~~...
```
- 增加新机制：洪水每回合从左侧和下方同时涌入
- 胜利条件：救下3人（不变），但增加"所有村民必须到达高地"的隐藏目标

**通关策略多样性**：
1. 造桥流：在(3,3)-(5,3)造桥直接通行
2. 吸水+桥：先吸水降低水位，再造桥
3. 冻结流：冻结水域让村民踩冰过去
4. 引导流：用引导让村民绕过水域
5. 高地流：抬升地面形成临时高地躲避洪水

### 第2关：永夜矿井
**问题**：照明效果被黑暗扩散抵消
**修改**：
- 黑暗扩散速度：2格/回合 → 1格/回合
- 增加新机制：矿工自带"矿灯"，每回合随机照亮1格周围
- 增加出口：从1个增加到2个（东北和西北）

**通关策略多样性**：
1. 照明流：大范围照明驱散黑暗
2. 引导流：用引导让矿工直接走向出口
3. 净化流：用净化永久清除黑暗（但消耗高）
4. 路径流：显示最优路径，让矿工快速移动
5. 森林流：种森林阻挡黑暗扩散

### 第3关：巨兽困城
**问题**：巨兽6回合正好到达城市，容错率低
**修改**：
- 增加回合数：6 → 8
- 增加新机制：巨兽有"饮水"需求，每2回合必须到达水域，否则会狂暴
- 地图上增加2格水域（不在直线上）

**通关策略多样性**：
1. 阻挡流：造墙挡住巨兽
2. 安抚流：降低怒气让巨兽减速
3. 陷阱流：设置陷阱让巨兽停留
4. 引导流：引导巨兽走向水域（拖延时间）
5. 迟缓流：直接让巨兽每回合只移动半格
6. 森林流：种森林让巨兽绕路

### 第4关：失语战争
**问题**：初始战争值太高（3/9）
**修改**：
- 初始战争值：3 → 1
- 增加新机制：使者每回合靠近1格，战争值+1（紧迫感）
- 增加"误会事件"：每2回合随机触发一个误会事件，战争值+1

**通关策略多样性**：
1. 安抚流：直接降低战争值
2. 沟通流：让使者在边境快速会合
3. 梦境流：用dream_link连接两个使者，共享视野
4. 净化流：净化迷雾让使者看清道路
5. 时间流：用时间延缓争取更多回合

### 第5关：记忆瘟疫
**问题**：村民路径被完全阻挡，0人获救
**修改**：
- 减少沼泽：从3格减少到2格
- 增加新机制：村民有"记忆碎片"，每回合有20%概率自动记起正确方向
- 增加"回响地形"：某些格子会重复上回合的地形效果

**通关策略多样性**：
1. 记忆信标流：放置记忆信标引导村民
2. 引导流：直接引导村民移动
3. 净化流：清除迷雾和沼泽
4. 森林流：种森林形成安全通道
5. 桥梁流：在沼泽上造桥
6. 路径流：显示最优路径，让村民自动跟随

### 第6关：终考
**问题**：多威胁同时出现，难以应对
**修改**：
- 增加造物次数：4 → 5
- 增加奇迹点：8 → 10
- 增加新机制：每回合只激活一种主要威胁（轮替：洪水→黑暗→战争→巨兽→循环）
- 增加"考核支援"：第4回合自动获得1次免费造物次数

**通关策略多样性**：
1. 综合流：每种威胁各用一个造物应对
2. 共鸣流：利用环境共鸣强化效果
3. 连锁流：利用连锁反应产生复合效果
4. 时间流：用时间延缓争取更多时间
5. 强化流：用强化模式快速解决威胁

## 实现计划

### Phase 1：新增能力（5个文件）
1. `aiClient.js`：增加新能力的关键词和名称种子
2. `debugGame.js`：实现新能力的游戏逻辑
3. `levels.js`：调整关卡地图和参数
4. `server.js`：更新AI编译器的能力枚举
5. `game.js`：同步前端游戏逻辑

### Phase 2：新机制（3个文件）
1. `debugGame.js`：实现环境共鸣、连锁反应
2. `game.js`：同步前端
3. `public/index.html`：增加UI提示（共鸣、连锁提示）

### Phase 3：副作用选择（2个文件）
1. `debugGame.js`：实现强化模式
2. `cli.js`：增加模式选择命令

### Phase 4：测试验证
1. 用CLI测试每个关卡的多种通关策略
2. 确保没有"不造任何东西就能通关"的情况
3. 确保每种策略都有可行性和代价
