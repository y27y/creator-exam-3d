# 造物者考核 3D：环境美术与地图优先界面重构设计

日期：2026-07-10
状态：已由用户确认，可进入实施

## 一句话方案

把现有“圆角玻璃后台”改成“裂隙战术档案”：棋盘和造物操作始终占据主视野，六关获得各自的实时环境舞台，固定剧情节点使用 4 张开发期生成的 CG，高级系统进入情境抽屉，重要的新传说等内容通过可点击世界信号主动提醒玩家。

## 背景与问题

当前项目的核心玩法已经完整，但呈现层有三个直接影响试玩体验的问题：

1. 主页面包含 17 个 `section` 和 1 个默认展开的测试沙盘；右侧长面板同时容纳造物、测试入口、居民对话、AI 战术室、传说、仪式、誓约、深渊、腐化、工坊和传承等内容。
2. 大面积圆角玻璃卡片视觉权重接近，关卡目标、核心操作和高级内容难以一眼区分；地图在左右面板之间被压缩。
3. 主流程缺少序章和关键过场美术；守城只有纯 CSS 三页过场；空战已有 70 张 PNG，但其高密度科幻风与主世界的低多边形棋盘缺少桥接。

现有空战图片共 50,898,132 bytes（约 48.54 MiB），且 `AirCombatAssets.load()` 当前会遍历整个 manifest 并在空战启动时全部加载。新增美术必须同时约束首屏体积与加载策略。

## 已确认的设计决定

- 总体路线：选择“裂隙神话混合风”。固定剧情使用电影感环境 CG，实时玩法保留清晰的低多边形棋盘。
- 造物边界：玩家造物由自然语言实时生成，不能预制逐件插画。造物继续按能力类型映射几何、材质、颜色、符文和粒子。
- 图像模型边界：运行时不调用 image-2。image-2 只在开发期生成固定剧情 CG。
- 主界面：选择“地图优先 · 情境抽屉”。核心操作常驻，高级系统按需打开。
- 重要信息：简化常驻内容，但新传说、居民请求和不可逆节点必须主动提示并可回看。
- 整体风格：选择“裂隙战术档案”——哑光深色、切角结构、暖金叙事、青绿交互、紫色裂隙。

## 目标

- 首次玩家在不滚动页面的情况下完成“理解目标 → 生成造物 → 放置 → 结束回合”。
- 棋盘成为视觉主角，桌面端主游戏区域明显大于两侧信息区域。
- 六关在不改变地图规则的前提下拥有可辨识的环境主题。
- 用 4 张固定 CG 建立序章、长夜、空域和结局的叙事节奏。
- 所有高级系统保留，但不再同时争夺玩家注意力。
- 新传说等重要内容会主动提醒、保留未读入口并能直接定位。
- 所有新增美术均可本地运行；素材缺失时核心游戏仍完整可玩。

## 非目标

- 不修改 `GameEngine`、能力数值、敌人规则、关卡胜负条件或 AI 生成逻辑。
- 不为任意自然语言造物生成或准备独立图片、模型。
- 不在运行时调用图像生成 API。
- 不引入 React、Vue、状态库、美术管理框架或新的后端服务。
- 不删除仪式、誓约、深渊、腐化、工坊、传承等已有玩法。
- 不使用远程图片热链，不让游戏运行依赖素材网站在线可用。
- 不把环境装饰加入碰撞、寻路、射线选格或胜负计算。

## 视觉架构

### 固定剧情层

固定剧情层只包含 4 张无内嵌文字的环境 CG。标题、玩家本局数据和按钮全部由 HTML 覆盖，避免生成图中文字错误，也方便复用与无障碍读取。

| 文件建议 | 比例 | 使用位置 | 构图要求 |
|---|---:|---|---|
| `cg-prologue.webp` | 16:9 | 首次进入、标题主视觉 | 裂隙世界与远处棋盘；人物只作小比例剪影；左右保留文字安全区 |
| `cg-night-watch.webp` | 21:9 | 长夜守城三页过场 | 一张横向长卷包含城墙、居民与六段夜色；三页通过平移和裁切复用 |
| `cg-airspace-bridge.webp` | 16:9 | 守城后进入空域前 | 地面环境被裂隙压缩到天空；只表现抽象奇迹光，不描画具体玩家造物 |
| `cg-ending.webp` | 16:9 | 最终清算 | 中性的世界远景；胜负和熵值通过色调、裂纹覆盖和动态文案表现 |

四张图使用 image-2 在开发期生成，再压缩为 WebP。它们不随玩家输入变化，也不需要网络请求。

### 六关实时环境层

环境装饰位于棋盘外圈、远景或天空，不覆盖格子，不接收选格射线。优先复用 Three.js 原生几何与现有材质缓存；只有外部模型或纹理能明显提升效果时才引入最小的 CC0 文件。

| 关卡 | 环境主题 | 视觉元素 | 主色 |
|---|---|---|---|
| 洪水村庄 | 被淹没的低地聚落 | 外圈水面、远处屋顶、斜雨、冷雾；高地使用暖色引导 | 冷青 + 暖金 |
| 永夜矿井 | 被黑暗吞噬的矿道 | 岩壁环、断轨、矿灯、吞光雾；出口是唯一稳定冷白光 | 炭黑 + 冷白 |
| 巨兽困城 | 河兽逼近的森林城墙 | 林木与城墙远景、巨大脚印剪影、河水反光 | 青绿 + 琥珀 |
| 失语战争 | 被雾切开的双边边境 | 两侧旗帜、边境石、中央雾带；不新增士兵模型 | 紫灰 + 暖黄 |
| 记忆瘟疫 | 名字正在消失的圣树 | 圣树冠层、漂浮姓名碎片、淡紫记忆雾；接近圣树逐渐变亮 | 青玉 + 淡紫 |
| 第七天之前 | 前五关残片汇聚的终考 | 前五关环境道具残片围成裂隙环，复用已有资源重组 | 深蓝 + 金紫 |

### 实时造物层

不尝试还原玩家句子里的具体名词，而是让能力结果可读：

| 能力族 | 视觉语法 |
|---|---|
| 水 | 透明体积、青蓝流线、涟漪 |
| 光 | 暖金光柱、指路纹、短脉冲 |
| 地形 | 石块、根系、叶片、隆起轮廓 |
| 防御 | 环墙、力场、陷阱刻线 |
| 精神 | 紫色丝带、记忆字形、连接线 |
| 特殊 | 白金裂纹、时间残影、反向粒子 |

玩家输入的造物名称和描述原样显示。`getCreationVisualStyle()` 与 `createCreationMesh()` 继续是统一入口，只扩充能力族形状与特效；不增加“每种造物一个类”或远程生成路径。

## 公开素材策略

优先来源均为官方页面明确声明 CC0 的目录：

- [Poly Haven 许可说明](https://polyhaven.com/license)：HDRI、纹理和 3D 模型。
- [ambientCG](https://ambientcg.com/)：纹理、HDRI 和模型。
- [Kenney 支持与许可](https://kenney.nl/support)：游戏 UI、粒子和低多边形资源。
- [Quaternius FAQ](https://quaternius.com/faq.html)：低多边形环境模型。
- [OpenGameArt FAQ](https://opengameart.org/content/faq)：只选择具体条目明确标注 CC0 的资源，不混用其余许可证。

实施时无需再次进行设计选择，可按以下规则直接挑选最小文件：

1. CC0 优先；不使用许可证只写“免费”或条款不清的素材包。
2. 下载后放入仓库，不使用外链。
3. 只取实际使用的单个资源，不整包搬入。
4. 每个外部资源在 `public/assets/art/ATTRIBUTION.md` 记录本地文件、来源 URL、作者、许可证、下载日期和修改方式。CC0 虽无需署名，仍保留来源以便审计。
5. 开发期生成的 4 张 CG 在同一文件中记录模型、生成日期、最终提示词和后处理方式。
6. 不使用素材站 Logo、预览缩略图或未包含在资产许可证内的页面内容。

## 地图优先的信息架构

### 首屏始终可见

- 顶部细条：关卡、回合、裂隙和必要资源。
- 左上任务卡：当前故事一句话、目标、关键进度。
- 中央棋盘：最大可用区域。
- 底部造物坞：自然语言输入、灵感、生成、当前卡牌、放置和结束回合。
- 右侧入口：`人`、`志`、`术` 三个情境抽屉按钮及未读数。

重开关卡和设置进入次级菜单，不与“结束回合”同权常驻。生成卡牌后，卡牌详情在造物坞上方临时展开，不长期占用右栏。

### 情境抽屉分组

| 抽屉 | 内容 |
|---|---|
| 人 | 当前单位、居民对话、群聊、社交关系、传承角色 |
| 志 | 敌方意图、世界传说、物件、历史人物、蝴蝶效应、回合记录、长夜与空域入口 |
| 术 | 讲述者设置、AI 战术室、仪式、誓约、认知深渊、验证腐化、工坊、存档 |

每次只打开一个抽屉。`Escape` 关闭抽屉并把焦点还给入口按钮。抽屉内部允许滚动，但核心回合流程不依赖抽屉滚动。

### 测试入口

`#test-jump-panel` 保留 DOM ID 和现有测试能力，但默认隐藏；只有 URL 含 `?debug=1` 时显示。这样现有自动化入口仍可使用，普通玩家不会看到考官沙盘。

## 世界信号与提示层级

现有 `showToast()` 和回合日志继续保留，但消失的 toast 不再是重要信息的唯一载体。

| 优先级 | 例子 | 呈现 | 是否暂停 |
|---|---|---|---|
| 轻提示 | 新传说、图鉴、记忆更新 | 可点击信号条 + 对应抽屉未读徽标 | 否 |
| 行动提示 | 居民求助、可用仪式、工坊机会 | 持久任务卡，直到处理或明确忽略 | 否 |
| 关键节点 | 胜负、裂隙临界、长夜、空域、不可逆选择 | 现有 modal 或固定 CG | 是 |

实现只需要 `CreatorExam3D` 内一个普通 helper，不增加通知类或新状态库：

```js
notifyDrawer('world', {
  id: myth.id,
  priority: 'ambient',
  title: '新的传说正在流传',
  summary: myth.text,
  targetId: 'legend-panel'
})
```

行为规则：

- 用稳定事件 ID 在当前会话去重。
- 同一回合的同类轻提示合并为一条摘要，未读数累加。
- 点击信号或徽标打开正确抽屉并定位目标条目。
- 打开目标后清除对应未读数。
- 未读徽标只表示本次会话尚未查看；传说、关系和系统结果仍由现有持久化数据保存，因此徽标重置不会丢失内容。
- 关键提示优先于轻提示；轻提示不得覆盖胜负按钮或造物坞。

## 整体视觉语言

移除“每块都是大圆角玻璃卡片”的统一处理，改成层级明确的裂隙档案界面。

### 色彩令牌

| 用途 | 色值 |
|---|---|
| 世界底色 | `#070a11` |
| 哑光面板 | `#151923` |
| 主文字 | `#f0e4c1` |
| 叙事／关键节点 | `#d2b86b` |
| 交互／安全／成功 | `#59c7a6` |
| 裂隙／精神／异常 | `#8f73c8` |
| 危险 | `#e87f72` |

### 形状与排版

- 核心卡片使用小圆角或切角，减少 18–28px 大圆角。
- 背景以不透明或高不透明度哑光色为主，减少大范围 `backdrop-filter`。
- 暖金只标识叙事和关键行动；青绿只标识可交互和安全反馈；紫色只标识裂隙异常。
- UI 继续使用本机系统无衬线字体，避免新增字体体积；只有 CG 大标题使用系统衬线回退。
- 动画发生在事件源附近，不让所有面板同时入场。
- 保留并扩展现有 `prefers-reduced-motion` 和高对比模式。

## 组件与文件职责

设计遵循现有结构，不增加 `ArtManager`、`NotificationSystem` 或 UI 框架。

| 文件／目录 | 责任 |
|---|---|
| `public/js/game.js` | 添加一个 `environmentGroup`、六关环境配置、`applyLevelEnvironment(level.id)`、抽屉切换和 `notifyDrawer()`；继续复用现有几何／材质缓存、`loadLevel()`、`showToast()`、`showModal()` |
| `public/index.html` | 把现有内容重新分组为任务卡、造物坞、情境入口和单一抽屉；保留原有重要 ID |
| `public/styles.css` | 新视觉令牌、地图优先布局、抽屉、信号条、响应式和降级样式 |
| `server.js` | 在现有 MIME 表中加入本地 WebP；若最终选用 GLB，再同时加入对应 MIME，不改路由结构 |
| `public/modes/tower-defense/towerBridge.js` | 给现有三页守夜过场增加同一张横向 CG 的三种裁切位置 |
| `public/modes/air-combat/airCombatAssets.js` | 从启动时全量加载改为当前阶段加载并预取下一阶段，避免新美术叠加到约 48.54 MiB 首载 |
| `public/assets/art/` | 4 张固定 CG、少量实际使用的 CC0 环境资源和 `ATTRIBUTION.md` |
| `debug/browser-demo-smoke.js` | 验证核心操作、抽屉、世界信号和 debug 入口 |
| `debug/browser-modes-smoke.js` | 验证守城 CG、空战按阶段加载和桥接结果 |

## 数据流

```text
现有 GameEngine / WorldLegend / NPC / 高级系统事件
  → CreatorExam3D 现有状态更新
  → updateUi / renderLegendPanel 等现有渲染入口
  → notifyDrawer（只保存 UI 未读与定位信息）
  → 信号条 / 未读徽标
  → 玩家打开情境抽屉
  → 复用现有 panel 内容与持久化数据
```

关卡环境的流向保持独立：

```text
loadLevel(index)
  → 现有 GameEngine 初始化
  → applyLevelEnvironment(level.id)
  → 清空 environmentGroup 并应用灯光、雾和外圈装饰
  → 现有 renderWorld 渲染地块、单位和实时造物
```

## 资源加载与性能

- 4 张固定 CG 使用 WebP；透明小图才保留 PNG。
- 新增视觉资源总目标不超过 10 MB。
- 序章图可预加载，其余 CG 到达前一节点时再加载。
- 每关只加载该关环境资源；换关时移除环境组并释放独占纹理／模型。
- 新增环境资源从本地加载，且不能阻塞 `loadLevel()`；程序化棋盘先可玩，装饰完成后再出现。
- 空战只加载当前世界需要的背景、Boss 和效果，预取下一世界；不再遍历整个 manifest 一次性加载。
- 不因美术改造提高粒子上限或设备像素比上限。

## 错误处理与降级

| 失败点 | 玩家看到的降级结果 | 处理 |
|---|---|---|
| CG 缺失或解码失败 | 同色渐变、裂纹纹理和完整 HTML 文字 | 立即显示按钮，过场继续 |
| 环境纹理／模型失败 | 当前程序化棋盘与默认灯光 | 记录一次警告，跳过该装饰 |
| 同回合提示过多 | “世界志新增 N 条”等摘要 | 同类合并，未读累加，关键事件优先 |
| 小屏 | 全屏抽屉和固定底部造物坞 | 核心按钮不随抽屉内容滚走 |
| 减少动态效果 | 取消平移、缩放和长转场 | 保留瞬时状态变化与文字反馈 |

任何美术失败都不得改变游戏状态、阻塞回合或阻止守城／空战桥接。

## 无障碍要求

- 三个抽屉入口使用真实 `button`，包含可读名称、`aria-expanded` 和未读说明。
- 世界信号使用 `aria-live="polite"`；胜负和不可逆选择继续使用对话框语义。
- 打开抽屉后焦点进入标题或第一可操作项；关闭后回到原入口。
- 所有文字覆盖与 CG 背景之间维持足够对比，必要时使用实色遮罩而不是只靠文字阴影。
- 键盘可以完成造物输入、生成、放置模式启动、结束回合、打开／关闭抽屉和过场继续。

## 测试与验收

### 自动回归

扩展现有 browser smoke，不新建测试框架：

- 核心造物输入、生成、放置和结束回合 ID 继续存在且可点击。
- 普通 URL 隐藏测试沙盘，`?debug=1` 显示它。
- `人`、`志`、`术` 三个抽屉均可打开，且同一时间只有一个展开。
- 新传说只产生一次轻提示，未读数增加，点击后定位传说并清零。
- 多个同类轻提示会合并，关键 modal 不被覆盖。
- 缺失 CG 和环境资源时 `loadLevel()` 仍完成，回合仍可结束。
- 守城上下文、空战上下文和结果回写保持原协议。
- 空战资产加载不再在启动时请求全部背景阶段。

相关验证命令：

```bash
npm run check
npm run test:reality
node debug/browser-demo-smoke.js
node debug/browser-modes-smoke.js
node debug/verify-all.js
```

### 视觉验收

- 在 1280×720 和 1440×900 下，任务、造物输入、生成、放置和结束回合无需页面滚动即可操作。
- 在宽度不超过 760px 时，抽屉改为全屏层，造物坞仍固定可用。
- 六关逐关截图确认环境装饰不遮格子、单位、路径和敌方意图。
- 高对比与减少动态效果下，所有核心状态仍可辨认。
- 序章、长夜、空域和结局 CG 均可跳过，按钮可键盘聚焦。

### 完成标准

1. 首次玩家能够在首屏完成一回合，无需理解高级系统。
2. 地图占据主视野，高级内容只在情境抽屉中出现。
3. 六关环境一眼可区分，但玩法和选格不受影响。
4. 4 张固定 CG 串起主线，不描画具体实时造物。
5. 新传说会主动提示、留下未读入口并可回看。
6. 关闭或删除所有新增美术文件后，核心游戏仍完整运行。
7. 所有自动回归与视觉断点检查通过。

## 风险与控制

- **风格不统一**：所有外部素材先经过同一色板、雾、粗糙度和轮廓处理；不直接把写实资产原样放入低多边形世界。
- **UI 简化导致功能难找**：通过情境信号、未读徽标和清晰的三类抽屉入口解决，不恢复长面板。
- **CG 暗示不存在的具体造物**：构图只使用环境、居民剪影、裂隙光和抽象奇迹，不出现可被误认为本局造物的固定物件。
- **资产体积继续膨胀**：只取单个文件、WebP 压缩、10 MB 新增上限、空战按阶段加载。
- **重排 DOM 破坏测试和调用**：保留现有关键 ID，以扩展现有 smoke 为提交前门槛。
