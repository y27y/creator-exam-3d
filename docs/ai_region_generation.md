# AI生成关卡系统架构

## 核心概念

**不是固定关卡，而是无限生成的"世界碎片"**

每个"关卡"是一个7x7的区域（世界碎片），由AI根据以下因素动态生成：
- 玩家之前的行为和选择
- 当前世界裂隙值
- 已发现的叙事线索
- 玩家与NPC的关系
- 前一片区域的残留影响

## 生成参数

### 1. 区域主题（Theme）
根据玩家行为倾向动态选择：
- 如果玩家喜欢用水系能力 → 生成水域主题区域
- 如果玩家裂隙值高 → 生成崩解主题区域
- 如果玩家救了很多人 → 生成希望主题区域

### 2. 地形生成（Terrain）
```javascript
{
  map: ['~~~....', '~~..~~~', '~..V..H', '~~~V~~H', '~~.V..H', '~~~....', '~~~~...'],
  // 不是固定的，而是根据主题生成
}
```

### 3. 单位生成（Units）
- 可能遇到之前救过的角色（他们记得你）
- 新角色可能与旧角色有关联
- 敌人的行为模式会根据你的策略调整

### 4. 危机生成（Hazard）
- 不是固定的"洪水""黑暗"，而是混合危机
- 危机强度根据玩家实力动态调整
- 危机可能与之前区域的选择有关

### 5. 目标生成（Objective）
- 不是固定的"救3人"，而是动态目标
- 目标类型：救援、探索、护送、谈判、建造
- 目标难度根据玩家历史表现调整

## 生成流程

```
玩家完成当前区域
  ↓
系统收集玩家数据（行为、选择、裂隙、关系）
  ↓
AI生成下一片区域的参数
  ↓
系统根据参数生成具体地图、单位、危机
  ↓
AI生成区域的故事背景和目标
  ↓
玩家进入新区域
```

## 实现方案

### 后端API

```javascript
// POST /api/generate-region
{
  playerState: {
    playStyle: 'creative',      // 玩家风格
    entropy: 3,                 // 当前裂隙
    rescued: 5,                   // 总救援数
    lost: 2,                    // 总损失数
    creations: ['月光水母', '藤蔓城垣'], // 用过的造物
    relationships: {            // 角色关系
      'old-fisherman': 'friendly',
      'village-child': 'loved'
    },
    discoveredLore: ['lore-1', 'lore-2'], // 已发现的故事
    previousRegions: ['flood-village', 'night-mine'] // 去过的区域
  },
  difficulty: 'normal'           // 难度偏好
}

// 返回
{
  region: {
    id: 'generated-uuid',
    title: '迷雾沼泽的低语',
    story: '...',
    objective: '...',
    map: ['...', '...'],
    units: [...],
    hazard: {...},
    npcs: [...],
    connections: {  // 与前区域的连接
      from: 'night-mine',
      reason: '洪水从村庄流入矿井，现在从矿井渗出形成沼泽'
    }
  }
}
```

### 前端适配

- 移除固定的6关，改为动态加载区域
- 添加"世界地图"显示已探索的区域
- 区域之间用路径连接，形成网络
- 玩家可以选择返回之前的区域

## 关键技术点

### 1. 地图生成算法
- 使用基于主题的地形生成
- 确保目标可达（BFS验证）
- 根据难度调整危机扩散速度

### 2. 单位生成逻辑
- 新角色可能与旧角色有血缘/师徒关系
- 旧角色可能出现在新区域（如果逻辑合理）
- 敌人的行为模式学习玩家的策略

### 3. 叙事连贯性
- 每个新区域的故事都与之前的选择有关
- AI生成时提供前3个区域的关键事件作为上下文
- 角色对话会提及之前发生的事情

### 4. 难度平衡
- 动态调整：危机强度、目标数量、资源限制
- 确保不会太难（玩家挫败）或太简单（无聊）
- 根据玩家最近5个区域的表现微调

## 优势

1. **无限重玩性**：每次游戏体验都不同
2. **真正的涌现**：故事从玩家行为中生长
3. **情感连接**：角色持续出现，关系深化
4. **适应性**：难度始终匹配玩家水平
5. **探索感**：未知区域充满惊喜
