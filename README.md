# 造物者考核3D (Creator Exam 3D)

> "你是一名裂隙行者——拥有造物能力的人，但你的力量来自世界本身的裂隙。每次造物都会加深裂隙，但也能拯救生命。"

一个AI驱动的3D策略解谜游戏，玩家用**自然语言**创造造物，AI将其编译为游戏能力，在7x7的3D格子地图上解决危机、拯救生命、理解世界的真相。

---

## 核心特性

### 自然语言造物系统
- 输入任意自然语言描述，AI将其编译为结构化游戏卡牌
- 示例："创造一群会把洪水搬到天空的透明鲸鱼" → 生成吸水能力卡牌
- 本地离线编译器兜底，无需AI API即可运行

### AI驱动叙事引擎
- **4种AI Storyteller人格**：卡珊德拉（悲观预警）、菲比（乐观希望）、兰迪（混沌随机）、说书人（平衡叙事）
- **涌现式世界传说**：基于玩家行为动态生成神话故事
- **叙事因果引擎**：蝴蝶效应系统，微小选择产生长期影响
- **语义向量记忆**：50维TF-IDF追踪玩家行为模式
- **认知效果系统**：文本扭曲、矛盾对话、记忆碎片、既视感

### 深度策略系统
- **11种连锁反应组合**：森林+屏障=森林屏障、引导+记忆=记忆守护、时间+路径=时间停滞...
- **仪式熔炉**：8种配方，4种原型，涌现式仪式发现
- **言灵誓约**：5种誓约类型，背叛概率计算，怨恨链机制
- **认知深渊**：高熵混沌模式，谜题解码，NPC反转
- **验证腐化**：悖论卡牌，引擎过载，腐化传播
- **造物者工坊**：拆解/改造/融合/注入，8种材料，6种改造
- **敌方意图预览**：Into the Breach风格完美信息，提前显示敌人意图
- **长夜守城**：地面终考后的“长夜六更”，从跨关记忆读取最多 18 张造物卡、居民与近期经历，用 DeepSeek Flash 等模型生成守夜塔名称、因果简报、三选一守夜加成、数值偏向、射程与技能说明
- **第七日裂隙空域**：守夜结算后的终局 Canvas 空战清算层；四页过场明确承接守夜主题、波数、受护居民、守夜誓约与塔计划，再把造物武器化并以 6 段 Boss 航线回写最终结局
- **第七日之后**：空域裁决后的五幕多时间线终章；从真实六关记录生成稳定档案号、六段失败回声与居民／造物／誓约锚点，玩家选择最终锚点后把当前世界的答案传播给其他时间线；30 张本地 image-2 CG 支持刷新恢复与世界志重播
- **完整教学战役**：覆盖六关地面主线、长夜守城和第七日空域；推荐造物使用诗性提示词，目标位置直接在地图高亮，“人 / 志 / 术 / 教”侧栏可随时折叠，31 个高级系统步骤会在真实状态中重复体验

### NPC与社交系统
- **动态情绪系统**：8种情绪状态，情绪传染机制
- **社交图谱**：6种关系类型，小团体检测，社交距离计算
- **单位传承**：Wildermyth风格，8种特质，4个传承等级
- **跨关卡持久化**：被救援单位在未来关卡重现，关系状态继承

### 3D视觉体验
- Three.js低多边形渲染，InstancedMesh优化
- **NPC身份化造型**：20 名关卡角色拥有不同体型、配色、头饰与标志性随身物；后续动态居民按稳定身份生成造型，跨关重现时不会随机“换脸”。
- **image-2 人物立绘**：20 名可交谈 NPC 各有一张本地透明立绘，同一张图同时用于中上方圆形状态头像和右下角轻量对话；遇难头像自动灰显，点击棋盘人物即可直接交谈，造物坞不会被关闭或遮蔽。
- 粒子特效系统（能力激活、救援/失败、环境效果）
- 屏幕特效（闪光、震动、暗角、颜色分级）
- 动态路径可视化
- **地图优先界面**：任务、棋盘与造物坞常驻；人物、世界志和高级造物术按需从情境抽屉打开，新传说提示锚定在“人 / 志 / 术”按钮旁，不与右下角人物交互重叠。
- **六关环境与固定过场**：每关都有独立的 image-2 背景、程序化 3D 氛围和 CC0 外圈模型；序章、长夜、空域和终局使用本地固定 CG，守夜塔与空域载体使用同色系本地 image-2 美术，素材缺失时自动回退到 CSS 渐变或程序化画面。
- **关卡化沙盘地貌**：洪水村庄使用湿润青灰底、淤泥、芦苇、分层暗水、涟漪和漂木；永夜矿井使用石墨地表、矿脉、轨道和碎岩；巨兽困城使用森林石庭、苔藓、根系、树干与树冠；失语战争使用灰褐无人区、道路、界桩和战壕痕迹；记忆瘟疫使用低饱和雾面、镜池和记忆碎片；终考使用紫灰裂隙地表、汇流裂纹和混合世界残片。
- **六幕章节开场**：六关各使用三张连续 image-2 画面，配合少量网页文字、淡入、慢推镜和进入棋盘前的画卷收束；首次进入自动播放，也可从 `?debug=1` 的考官沙盘重播当前关。
- **本地声音连续性**：按钮、造物编译、落地、新传说与胜负使用本地 CC0 音效；主游戏、长夜守城和裂隙空域各自使用本地配乐承接终局气氛，全部只在首次玩家操作后解锁，并共享“更多”中的静音偏好。

## 快速开始

### 环境要求
- Node.js >= 18
- 现代浏览器（支持ES Modules和WebGL）

### 安装运行

```bash
# 克隆项目
cd creator_exam_3d

# 启动服务器
npm start

# 打开浏览器访问
http://localhost:3000
```

普通玩家入口不会显示考官沙盘。需要关卡跳转、守城／空战预演或浏览器烟测时，使用 `http://localhost:3000/?debug=1`。
章节开场可用 `http://localhost:3000/?debug=1&chapter=1` 强制重播；把最后的数字改为 `1` 到 `6` 可直接预览对应关卡。

首次进入可以选择完整教学。教学只显示当前一步；点击右侧“教”可展开或收起，造物落点由棋盘金色标记指示。守夜和空战也各有可折叠“教”按钮；若玩家实在无法通过，可使用仅在教学战斗中出现的“教学保底”外挂，正常黄金路线与自动验收不依赖该按钮。

正常终局顺序固定为“地面终考 → 长夜六更 → 第七日裂隙空域 → 最终结局”。非调试入口不能跳过守夜直接进入空域；最终结局只在空战结果回写后生成。

固定 CG、六关背景和 CC0 纹理位于 `public/assets/art/`，CC0 模型与音效分别位于 `public/assets/models/`、`public/assets/audio/`。完整来源记录见 `public/assets/ATTRIBUTION.md` 与 `public/assets/art/ATTRIBUTION.md`；运行时不会调用图像生成服务，也不会热链第三方素材。

> 项目无npm依赖，`npm start` 直接执行 `node server.js`。Three.js通过CDN加载。

教学黄金路线可单独验证：`npm run test:tutorial`。完整系统回归仍使用 `node debug/test-suite.js`。

### 接入AI（可选）

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
AI_API_KEY=你的密钥
AI_BASE_URL=https://你的-openai-compatible-endpoint/v1
AI_MODEL=你的模型名
PORT=3000
```

DeepSeek Flash 等 OpenAI-compatible 模型可通过 `AI_BASE_URL` / `AI_MODEL` 接入同一套代理。空战、NPC对话和主线叙事共用 `/api/narrative`；长夜守城额外使用 `/api/night-watch-towers`，由此前造物、居民、近期经历、熵值和守夜结果提供上下文。未配置密钥时，该端点使用本地兜底生成不同造物对应的守夜塔、因果说明和三选一守夜加成。

未配置AI时，游戏使用本地离线编译器和本地叙事兜底，仍可完整运行。

## 玩法说明

1. **观察**左侧关卡目标和当前危机状态
2. **创造**在右侧输入造物想法，例如：
   - "造一棵会发光并指路的月亮树"
   - "让两族的梦境在同一张桌子上相遇"
   - "创造一面吸收所有悲伤的镜子"
3. **生成**点击"生成造物卡"，AI编译为游戏能力
4. **放置**点击"放置造物"，在3D地图上选择格子
5. **结算**点击"结束回合"，观察灾害扩散、单位移动和连锁反应
6. **目标**达成关卡目标进入下一关，失败则重来

### 能力池
- **水系**：吸水、造桥、冻结
- **光系**：照明、路径显示、太阳祝福
- **地形系**：改造、抬升、种植森林、挖掘水渠
- **防御系**：墙壁、力场、陷阱
- **精神系**：安抚、引导、记忆信标、梦境连接
- **特殊**：净化、时间延缓

## 核心系统详解

### 1. AI Storyteller ( storyteller.js )
参考RimWorld的AI故事叙述者，4种人格动态调整游戏事件：
- **卡珊德拉**：事件频率递增，制造持续压力
- **菲比**：事件间隔较长，给予玩家喘息空间
- **兰迪**：完全随机，可能连续灾难或长期和平
- **说书人**：平衡型，根据玩家表现动态调整

### 2. 连锁反应系统 ( chainReactionCodex.js )
11种能力组合产生额外效果：
| 组合 | 效果 |
|------|------|
| 森林 + 屏障 | 森林屏障持续时间延长 |
| 引导 + 记忆信标 | 被引导单位免疫混乱 |
| 陷阱 + 迟缓 | 巨兽额外停留 |
| 冻结 + 抬升 | 高地永久存在 |
| 冻结 + 森林 | 形成霜木林 |
| 引导 + 梦境 | 获得梦境守护 |
| 力场 + 安抚 | 和平领域 |
| 改造 + 森林 | 改造地形直接变成森林 |
| 日华 + 抬升 | 高地周围黑暗永久驱散 |
| 净化 + 记忆信标 | 迷失单位找回方向 |
| 时间延缓 + 显路 | 造物持续时间暂停 |

### 3. 敌方意图预览 ( enemyIntent.js )
Into the Breach风格：每回合开始前显示所有敌人的意图：
- 巨兽：前进/冲锋/暴走/受阻/被困
- 平民：徘徊/前进/受引导/混乱/梦境连接
- 使者：前进/误判/受引导
- 灾害：扩散/加剧/休眠

### 4. 认知深渊 ( cognitiveAbyss.js )
高熵游戏层，当裂隙值超过阈值时触发：
- 谜题解码（凯撒密码、反向文本、符号替换）
- NPC反转（敌人变友军）
- 悖论卡牌（非法能力产生意外效果）
- 深渊深度系统（越深入奖励越高风险越大）

### 5. 造物者工坊 ( creatorWorkshop.js )
关卡之间的资源管理空间：
- **拆解**：将造物还原为材料
- **改造**：消耗材料改变造物属性（范围+1、持续时间+2等）
- **融合**：两个造物合成新造物（特殊配方产生稀有能力）
- **注入**：将传承单位的特质注入造物
